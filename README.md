# Ansible playbooks

Данные плейбуки используются для сбора информации и обновления модулей на 50+ разнообразных серверах Windows и Linux.  
В плейбуках реализовано формирование и отправка сообщений в Telegram-каналы.  
Все файлы инвентаря защищены с помощью **Ansible Vault**.  
Работа с плейбуками также настроена через **Ansible Semaphore**.

---

## 📁 Директория `dotnet`

Используется для проверки версий `.NET` и их обновления.

- `check_net.yml` — проверка версий .NET
- `update_dotnet.yml` — обновление .NET

---

## 📁 Директория `download`

Используется для обновления внутренних модулей: как с помощью архивов, так и отдельных `.exe`-файлов.  
После завершения обновления отправляется уведомление в Telegram с указанием обновлённого файла и его версии.  
Для архивов дополнительно указывается список файлов, находящихся внутри.

Плейбуки, выполняющие обновление без архива, предварительно проверяют наличие целевого файла перед обновлением.

- `CDA.yml` — обновление с помощью архива в директорию `typeplan`
- `CGI_BIN.yml` — обновление с помощью архива в директорию `cgi-bin`
- `JS.yml` — обновление с помощью архива в директорию `js`
- `JSON.yml` — обновление с помощью архива в директорию `json`
- `PNG.yml` — обновление с помощью архива в директорию `images`
- `create_derectory.yml` — создание папки
- `hostplaybook_dispcgi.yml` — обновление модуля `dispcgi`
- `hostplaybook_myloadfile.yml` — обновление модуля `myloadfile`
- `hostplaybook_nascgi.yml` — обновление модуля `nascgi`
- `hostplaybook_odii.yml` — обновление модуля `odii`
- `hostplaybook_palliat.yml` — обновление модуля `palliat`
- `hostplaybook_rest_tcgi.yml` — обновление модуля `rest_tcgi`
- `hostplaybook_sendgosp_regis.yml` — обновление модуля `sendgosp_regis`
- `hostplaybook_setup_et.yml` — обновление модуля `setup_et`
- `hostplaybook_setup_s_et.yml` — обновление модуля `setup_s`
- `hostplaybook_statcgi.yml` — обновление модуля `statcgi`
- `hostplaybook_svorthanc.yml` — обновление модуля `svorthanc`
- `hostplaybook_tcgi.yml` — обновление модуля `tcgi`
- `loadterm_oid.yml` — с помощью приложения `loadterm_oid` скачиваются справочники из сервиса терминологии (перечислены в переменной `oid_parameters`), архивируются и отправляются на FTP, после чего выполняется обновление по всем точкам. Запускается автоматически через **Cron**.

## 📁 Директория `lis`

Используется для сбора данных о файлах, которые не были отправлены из-за ошибок на серверах.

- `OdliSendResults.yml` — обновление модуля `OdliSendResults` с помощью архива
- `find_req_bad.yml` — сбор данных с серверов о количестве файлов с расширениями `.req` и `.bad` с последующим уведомлением в Telegram. Также указываются недоступные точки. Запускается автоматически через **Cron**.

## 📁 Директория `sendedapi_test`

Используется для обновления модулей `sendedapi`, `sapart`, а также для удаления базы `cache`.

- `sendedapi.yml` — обновление модуля `sendedapi` с помощью архива
- `sendedapi_delete_cache.yml` — удаление базы `cache`. Запускается автоматически через **Cron**
- `sendedapi_dop.yml` — обновление модуля `sapart` с помощью архива
